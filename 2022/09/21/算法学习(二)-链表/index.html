三、设计链表（707）

leetcode题目链接：https://leetcode.cn/problems/design-linked-list/

题目描述：

​ 设计链表的实现。您可以选择使用单链表或双链表。单链表中的节点应该具有两个属性：val 和 next。val 是当前节点的值，next 是指向下一个节点的指针/引用。如果要使用双向链表，则还需要一个属性 prev 以指示链表中的上一个节点。假设链表中的所有节点都是 0-index 的。

在链表类中实现这些功能：

get(index)：获取链表中第 index 个节点的值。如果索引无效，则返回-1。
addAtHead(val)：在链表的第一个元素之前添加一个值为 val 的节点。插入后，新节点将成为链表的第一个节点。
addAtTail(val)：将值为 val 的节点追加到链表的最后一个元素。
addAtIndex(index,val)：在链表中的第 index 个节点之前添加值为 val 的节点。如果 index 等于链表的长度，则该节点将附加到链表的末尾。如果 index 大于链表长度，则不会插入节点。如果index小于0，则在头部插入节点。
deleteAtIndex(index)：如果索引 index 有效，则删除链表中的第 index 个节点。

七、环形链表II（142）

leetcode题目链接：https://leetcode.cn/problems/linked-list-cycle-ii/

题目描述：

​ 给定一个链表，返回链表开始入环的第一个节点。 如果链表无环，则返回 null。

为了表示给定链表中的环，使用整数 pos 来表示链表尾连接到链表中的位置（索引从 0 开始）。 如果 pos 是 -1，则在该链表中没有环。不允许修改给定的链表。

​ 本题细分下来会发现有两个问题：1.判断链表内是否有环？2.链表环的入口在哪？

​ 第一个问题很好解决，定义两个指针fast和slow，fast每次走两步，slow每次走一步，如果fast和slow相等了，就说明有环。（因为fast相对于slow的速度是1，所以fast不可能跨过slow）

​ 第二个问题相对较难，环的入口在哪？根据分析，我们可以获得以下关系：

又因为fast走过的路程是slow 的两倍，假设fast在里面转了n圈，则：x+y+n(y+z)=2(x+y)，可以得到x=n(y+z)-y，再化简得，x=(n-1)(y+z)+z.

​ 再进行分析，当slow走了一圈时，fast已经走了两圈，这说明不管起点在哪，当第一次相遇时，slow一定是一圈没有走完，而fast已经走完了一圈，也就是说可以得到n=1，代入x=(n-1)(y+z)+z中得，x=z，也就是说如果一个指针从起点开始，另一个指针从fast与slow第一次相遇的节点开始，两指针以相同的速度开始运动，最终的交点就是环的入口！由以上分析就可以写出代码。
